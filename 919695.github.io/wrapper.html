python装饰器深入理解，爱上面向切面编程（一）

2016/1/9 posted in  python

装饰器模式是一个很著名的设计模式，经常被用于有切面需求的场景，较为经典的有插入日志、性能测试、事务处理等。装饰器是解决这类问题的绝佳设计，有了装饰器，我们就可以抽离出大量函数中与函数功能本身无关的雷同代码并继续重用。概括的讲，装饰器的作用就是为已经存在的对象添加额外的功能。

实现一个简单的装饰器

# 一个普通的函数
def text():
    return 'hello world!'


# 一个简单的装饰器 把要装饰的函数传递到新函数里构建一个新函数并返回,也就是闭包
def html(fn):
    def _wrapper():
        txt = '<b>{0}</b>'.format(fn())
        return txt

    return _wrapper

# 运行原始函数
txt = text()
print('原函数输出:', txt)


# 使用装饰器生产一个新函数
text = html(text)
txt = text()
print('新函数输出:', txt)

输出结果：

原函数输出: hello world!
新函数输出: hello world!

可以看到因为装饰了html()函数，新函数的输出结果被加粗了。装饰器在python里就是如此简单。
但是这这么复杂的写法好像并不怎么方便。

使用python的装饰器语法糖

# 一个简单的装饰器 把要装饰的函数传递到新函数里构建一个新函数并返回,也就是闭包
def html(fn):
    def _wrapper():
        txt = '<b>{0}</b>'.format(fn())
        return txt

    return _wrapper

# 一个使用了装饰器的函数
@html
def text():
    return 'hello world!'

txt = text()
print(txt)

输出结果：

hello world!

是不是特别爽，接下来只要函数是返回字符串，并且我想给他加粗，只需要再他上面标记@html就可以了，但实际工作中我们的函数要复杂的多。

实现函数带参数的装饰器

# 带参数函数装饰器
def html(fn):
    def _wrapper(author):
        txt = fn(author)
        txt = '<b>{0}</b>'.format(txt)
        return txt

    return _wrapper

# 标记装饰器
@html
def text(author):
    return 'hello world!' + '\nby:' + author

txt = text('wangke')
print(txt)

实现函数参数不确定的装饰器

# 参数不确定的函数装饰器
# 参数用(*args, **kwargs)，自动适应变参和命名参数
def html(fn):
    def _wrapper(*args, **kwargs):
        txt = fn(*args, **kwargs)
        txt = '<b>{0}</b>'.format(txt)
        return txt

    return _wrapper

# 标记装饰器
@html
def text(author, source):
    return 'hello world!' + '\nby:' + author + '\nsource:' + source


txt = text('wangke', source='http://wangke.pw')
print(txt)

输出结果：

hello world!
by:wangke
source:http://wangke.pw

实现起来不错，不过看上去哪里好像有问题，对了，我想加粗的只有内容，而作者和来源我并不想加粗。

让装饰器本身带参数

# 让装饰器带参数,在装饰器函数上再做一层装饰,
def html(author, source):
    def _wrapper(fn):
        def __wrapper(content):
            txt = fn(content)
            content = '<b>{0}<b/>'.format(txt)
            txt = '{0}\nauthor:{1}\nsource:{2}'.format(content, author, source)
            return txt
        return __wrapper
    return _wrapper


# 标记装饰器,并在装饰器传入参数
@html(author='wangke', source='http://wangke.pw')
def text(content):
    return content

txt = text('深入理解python装饰器')
print(txt)

输出结果：

深入理解python装饰器
author:wangke
source:http://wangke.pw

满足我们的需求了，到这里我们基本上也能熟练的使用装饰器了，但可能我们还会遇到一些问题，因为使用装饰器后已经完构建了一个全新的函数，而函数的原有属性则不存在了，我们试试分别输出标记@html装饰器前后的text函数的text.__name属性

标记前：__wrapper
标记后：text

在某些情况下我们不希望破坏原函数的属性，比如反射、单元测试，那么我们可以在装饰函数传递过程中把原函数属性赋值给新函数属性即可，但每次都去写赋值代码就比较麻烦了。

使用python内置的wraps函数

__author__ = 'wangke'

from functools import wraps

# 让装饰器带参数,在装饰器函数上再做一层装饰,
def html(author, source):
    def _wrapper(fn):
        # 使用warps装饰器对装饰器进行装饰
        @wraps(fn)
        def __wrapper(content):
            txt = fn(content)
            content = '<i>{0}<i/>'.format(txt)
            txt = '{0}\nauthor:{1}\nsource:{2}'.format(content, author, source)
            return txt

        return __wrapper

    return _wrapper

@html(author='wangke', source='http://wangke.pw')
def text(content):
    return content

txt = text('深入理解python装饰器')
print('属性name:', text.__name__)
print('-' * 20)
print(txt)

输出结果：

属性name: text
--------------------
深入理解python装饰器
author:wangke
source:http://wangke.pw

可以看到虽然使用了@html装饰器，但是text函数的原有属性仍然保留了，为了避免自己写的装饰器出现兼容性问题，我们应该把装饰器函数标记@wraps装饰器作为编码习惯。

标记多个装饰器

理解了装饰器就不难想明白，装饰器必然可以再包含装饰器，我们也就可以叠加标记装饰器。我们再写一个统计函数执行耗时的装饰器来叠加。

__author__ = 'wangke'

from functools import wraps

# 让装饰器带参数,在装饰器函数上再做一层装饰,
def html(author, source):
    def _wrapper(fn):
        # 使用warps装饰器对装饰器进行装饰
        @wraps(fn)
        def __wrapper(content):
            txt = fn(content)
            content = '<i>{0}<i/>'.format(txt)
            txt = '{0}\nauthor:{1}\nsource:{2}'.format(content, author, source)
            return txt

        return __wrapper

    return _wrapper

# 一个万能的可以统计函数执行耗时的装饰器
def consuming_time(fn):
    import time

    @wraps(fn)
    def _wraper(*args, **kwargs):
        start = time.time()
        time.sleep(1)
        result = fn(*args, **kwargs)
        end = time.time()
        print('执行耗时:', end - start)
        print('-----------------------')
        return result

    return _wraper
    
# 标记了多个装饰器，还可以继续标记
@consuming_time
@html(author='wangke', source='http://wangke.pw')
def text(content):
    return content

txt = text('深入理解python装饰器')
print('属性name:', text.__name__)
print('-' * 20)
print(txt)

输出结果：

执行耗时: 1.003476858139038
-----------------------
属性name: text
--------------------
深入理解python装饰器
author:wangke
source:http://wangke.pw

你已经学会python的装饰器了，发挥你的想象去实现自己的装饰器吧，要注意的是，在标记多个装饰器时要注意执行顺序是由下往上的。
